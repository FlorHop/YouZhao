# 有招蓝图展示功能 PRD

## 1. 文档信息

- 产品名称：有招
- 模块范围：蓝图展示、系统设置、用户管理、权限管理、MCP 服务
- 文档版本：v0.1
- 当前阶段：MVP 功能设计
- 前端约束参考：[前端统一规则约束](./frontend-rules.md)

命名约定：

- 产品和用户界面统一使用“蓝图”。
- 现阶段部分内部兼容标识仍使用 `demo`，包括权限键 `demo-preview`、历史数据字段和旧路由 `/demo`。
- 用户访问路由使用 `/blueprints`，旧路由 `/demo` 仅保留重定向兼容。

## 2. 背景与目标

有招需要先从“已部署蓝图的展示、授权、预览”开始实践 MVP。平台面向内部交付、售前、业务演示等场景，用户可以在统一入口查看已部署蓝图，并按权限打开对应版本的蓝图。

本阶段目标：

- 提供统一蓝图入口。
- 支持蓝图按分组展示和管理。
- 支持蓝图卡片基础信息展示。
- 支持蓝图版本选择和新页签预览。
- 支持用户管理。
- 支持功能权限和蓝图访问权限管理。
- 支持 Agent 通过 MCP 服务获取蓝图产物。
- 支持 Agent 通过 MCP 服务发布蓝图版本。
- 明确查看权限与管理权限的操作边界。

## 3. 用户角色

| 角色 | 说明 |
| --- | --- |
| 平台管理员 | 拥有系统设置管理权限，可管理用户、功能权限、蓝图权限。 |
| 蓝图管理者 | 拥有蓝图预览管理权限，可维护蓝图分组、卡片信息、卡片归属分组。 |
| 蓝图查看者 | 拥有蓝图预览查看权限，可查看被授权的蓝图，选择版本并打开预览。 |

说明：

- MVP 阶段不单独设计角色模型，权限直接授权到用户。
- 后续可扩展为“用户 - 角色 - 权限”的 RBAC 模型。

## 4. 功能范围

### 4.1 本期范围

- 左侧功能栏新增“蓝图”菜单。
- 蓝图页面以卡片形式展示已部署蓝图。
- 蓝图卡片按分组展示。
- 蓝图页面支持新建、删除分组。
- 删除分组时，将该分组下卡片移动至“默认”分组。
- “默认”分组不可删除。
- 分组支持排序，默认分组固定置顶。
- 蓝图卡片展示名称、概述、标签。
- 蓝图卡片支持拖动改变分组。
- 蓝图卡片支持右键菜单修改卡片信息和分组。
- 蓝图卡片支持版本下拉选择，默认最新版本。
- 点击蓝图卡片在浏览器新页签打开所选版本蓝图。
- 系统设置包含用户管理和权限管理。
- 用户管理支持新建、修改、删除。
- 权限管理支持功能权限授权。
- 权限管理支持按分组或单个蓝图授权。
- 新建用户默认拥有蓝图预览查看权限。
- 新建用户默认拥有“默认”分组的蓝图展示权限。
- 提供 MCP 服务，供 Agent 获取蓝图 HTML 与 Markdown 产物。
- 提供 MCP 服务，供 Agent 发布新的蓝图版本。

### 4.2 暂不纳入本期

- 蓝图上传、构建、发布流水线。
- 蓝图运行态内的细粒度页面权限。
- 角色管理。
- 组织架构管理。
- 单点登录。
- 审批流。
- 操作审计详情页。
- 蓝图使用统计分析。
- 蓝图全文搜索。

说明：本期不建设完整构建流水线，但允许 Agent 通过 MCP 提交已生成的 HTML 与 Markdown 产物，并由平台登记为蓝图版本。

## 6. 导航与页面

### 6.1 左侧功能栏

左侧功能栏包含：

| 菜单 | 显示条件 | 路由建议 |
| --- | --- | --- |
| 蓝图| 用户拥有蓝图预览查看或管理权限 | `/blueprints` |
| 系统设置 | 用户拥有系统设置管理权限 | `/settings` |

菜单规则：

- 用户没有任何可访问菜单时，显示无权限页。
- 系统设置仅有管理权限，不提供查看权限。
- 蓝图菜单内的数据仍需按蓝图权限过滤。

### 6.2 蓝图页面

页面目标：让用户快速查看、选择版本并打开已授权蓝图。

页面布局：

```text
PageHeader:蓝图
Toolbar: 新建分组 / 刷新 / 视图设置
ContentRegion:
  GroupSection: 默认
   蓝图Card[]
  GroupSection: 其他分组
   蓝图Card[]
```

显示规则：

- 仅展示当前用户被授权的分组和蓝图。
- 分组为空时仍可展示分组标题和空状态。
- 新建分组后即使为空也必须展示，不因没有蓝图卡片而隐藏。
- 默认分组始终存在。
- 蓝图卡片按分组展示。
- 分组顺序由管理者通过上移/下移调整，默认分组固定置顶。

管理能力：

- 拥有蓝图预览管理权限时，显示新建分组入口。
- 拥有蓝图预览管理权限时，显示分组删除入口。
- 拥有蓝图预览管理权限时，允许拖动卡片改变分组。
- 拥有蓝图预览管理权限时，允许通过右键菜单修改卡片内容和分组。
- 仅拥有蓝图预览查看权限时，不允许修改分组、名称、概述、标签。

查看能力：

- 拥有蓝图预览查看权限时，可以查看已授权蓝图。
- 版本下拉选择不属于数据修改类操作，查看权限用户也可使用。
- 点击卡片可新页签打开当前选择版本。

### 6.3 系统设置页面

系统设置包含两个模块：

- 用户管理
- 权限管理

显示规则：

- 仅拥有系统设置管理权限的用户可访问。
- 未授权用户访问时显示无权限页。

## 7. 蓝图展示功能

### 7.1 蓝图卡片

卡片字段：

| 字段 | 说明 | 是否必填 |
| --- | --- | --- |
| 蓝图名称 | 当前蓝图的展示名称 | 是 |
| 概述 | 蓝图的简短说明 | 否 |
| 标签 | 用于标识行业、场景、技术、状态等 | 否 |
| 版本 | 当前可预览版本列表 | 是 |
| 当前选择版本 | 卡片当前选中的版本 | 是 |

卡片操作：

| 操作 | 权限要求 | 说明 |
| --- | --- | --- |
| 选择版本 | 蓝图预览查看或管理 | 不修改蓝图数据，仅影响本次打开版本。 |
| 点击打开 | 蓝图预览查看或管理 + 蓝图授权 | 新页签打开所选版本蓝图。 |
| 拖动到分组 | 蓝图预览管理 | 修改蓝图所属分组。 |
| 右键编辑信息 | 蓝图预览管理 | 修改名称、概述、标签。 |
| 右键移动分组 | 蓝图预览管理 | 修改蓝图所属分组。 |

交互规则：

- 卡片点击区域用于打开蓝图。
- 版本下拉点击不触发卡片打开。
- 右键卡片打开上下文菜单，不触发卡片打开。
- 拖动卡片时不触发卡片打开。
- 拖动卡片但目标分组与原分组一致时，不产生修改动作，不提示“分组已变更”。
- 拖动分组变更需要后端接口确认成功后再提示成功；前端可先做乐观更新，但接口失败必须回滚。
- 调整分组顺序需要后端接口确认成功后再更新页面顺序。
- 修改成功后展示 Toast。
- 修改失败时展示 Toast，并保持原状态。

### 7.2 版本选择

版本规则：

- 每个蓝图至少有一个已部署版本。
- 默认选中最新版本。
- 最新版本以版本创建时间或部署时间倒序确定。
- 用户切换版本后，只影响当前页面会话中的选中值。
- 刷新页面后重新默认最新版本，除非后续实现用户偏好保存。

打开规则：

- 点击卡片时，打开平台蓝图预览容器页，而不是直接打开蓝图 HTML 产物。
- 平台预览容器页通过 iframe 加载真实蓝图 HTML 产物。
- 真实蓝图 HTML 产物保持纯净，不嵌入平台悬浮卡片、版本切换、权限等控制能力。
- 若版本不可用，提示“当前版本不可用”，不打开新页签。

### 7.2.1 平台预览容器

访问路径：

```text
/blueprints/{blueprintId}/preview?version={version}
```

页面结构：

```text
BlueprintPreviewPage
├─ iframe: 蓝图 HTML 产物
├─ FloatingBlueprintCard
│  ├─ 蓝图名称
│  ├─ 当前版本下拉
│  ├─ 文本内容入口
│  └─ 折叠/展开
└─ MarkdownDrawer
   └─ 蓝图 Markdown 文本内容
```

悬浮卡片规则：

- 悬浮卡片由平台预览容器渲染，不写入蓝图 HTML 文件。
- 悬浮卡片固定在左下角，尺寸小巧，不影响蓝图主体预览。
- 悬浮卡片必须支持版本下拉切换。
- 切换版本时仅切换 iframe 的产物地址，悬浮卡片不消失。
- 悬浮卡片提供“文本内容”入口，展示当前版本 Markdown 文本。
- 悬浮卡片可折叠。

产物规则：

- 每个蓝图版本包含 HTML 产物和 Markdown 产物。
- HTML 产物用于 iframe 预览。
- Markdown 产物用于蓝图文本内容展示和 Agent 获取。
- 平台预览地址由后端返回，前端不得直接拼接生产地址。

### 7.3 标签

标签用于提升识别效率。

规则：

- 标签展示在卡片正文区域。
- 单张卡片最多显示 5 个标签。
- 超出标签折叠为 `+N`。
- 标签内容由蓝图管理者维护。

### 7.4 分组管理

分组字段：

| 字段 | 说明 |
| --- | --- |
| 分组 ID | 系统生成，唯一 |
| 分组名称 | 页面展示名称 |
| 是否默认分组 | 标识默认分组 |
| 排序值 | 控制分组展示顺序 |
| 创建时间 | 分组创建时间 |

新建分组：

- 入口：蓝图页面 Toolbar。
- 权限：蓝图预览管理。
- 分组名称必填。
- 分组名称在当前租户或平台范围内不可重复。
- 新建成功后分组出现在蓝图页面。

删除分组：

- 入口：分组标题区域。
- 权限：蓝图预览管理。
- 默认分组不可删除。
- 删除非默认分组时需要二次确认。
- 删除后，该分组下所有蓝图卡片移动至“默认”分组。
- 删除成功后展示 Toast。

删除确认文案：

```text
删除分组后，该分组下的蓝图将移动至“默认”分组。是否继续？
```

调整顺序：

- 入口：分组标题区域。
- 权限：蓝图预览管理。
- 操作方式：上移、下移。
- 默认分组固定置顶，不可上移或下移。
- 查询或筛选状态下不展示排序操作，避免局部列表排序造成歧义。
- 顺序保存成功后，所有用户按新的分组顺序展示。

## 8. 用户管理

用户列表字段：

| 字段 | 说明 |
| --- | --- |
| 用户 ID | 系统主键，不可修改 |
| 用户名 | 登录或展示名称 |
| 姓名 | 用户真实姓名 |
| 邮箱 | 联系邮箱 |
| 手机号 | 联系手机号 |
| 状态 | 启用、停用 |
| 创建时间 | 用户创建时间 |

功能：

| 功能 | 说明 |
| --- | --- |
| 新建用户 | 创建用户，并自动赋予默认权限。 |
| 修改用户 | 可修改除用户 ID 外的信息。 |
| 删除用户 | 删除用户及其权限关系。 |

新建用户默认权限：

- 开放蓝图预览功能的查看权限。
- 开放蓝图展示功能中的“默认”分组权限。

删除规则：

- 删除用户需要二次确认。
- 删除用户后，该用户不可再登录或访问平台。
- MVP 阶段可采用硬删除；如后续需要审计，改为停用优先。

## 9. 权限管理

### 9.1 权限模型

MVP 阶段采用用户直接授权模型。

```text
User
├─ FunctionPermission[]
└─蓝图Permission[]
```

### 9.2 功能权限

功能板块：

| 功能板块 | 查看 | 管理 |
| --- | --- | --- |
| 蓝图预览 | 支持 | 支持 |
| 系统设置 | 不支持 | 支持 |

权限说明：

- 蓝图预览查看：可访问蓝图页面、查看已授权蓝图、选择版本、打开蓝图。
- 蓝图预览管理：包含查看能力，并可新建/删除分组、修改蓝图信息、修改蓝图分组。
- 系统设置管理：可访问系统设置、管理用户、管理功能权限、管理蓝图权限。

查看权限不可执行的数据修改类操作：

- 新建分组。
- 删除分组。
- 拖动蓝图卡片改变分组。
- 通过右键菜单修改蓝图名称、概述、标签。
- 通过右键菜单修改蓝图所属分组。

查看权限允许执行的非数据修改操作：

- 选择蓝图版本。
- 打开蓝图。

### 9.3 蓝图权限

授权形式：

| 授权方式 | 说明 |
| --- | --- |
| 按分组授权 | 用户可访问该分组下蓝图。 |
| 按单个蓝图授权 | 用户可访问指定蓝图。 |

权限合并规则：

- 用户最终可见蓝图= 已授权分组下蓝图∪ 已授权单个蓝图。
- 若蓝图同时被分组授权和单独授权，不重复展示。
- 用户有蓝图预览功能权限，但无任何蓝图权限时，进入蓝图页面显示空状态。

分组授权规则：

- 新建用户默认授权“默认”分组。
- 删除分组后，原分组授权关系失效。
- 被删除分组下蓝图移动至默认分组后，拥有默认分组权限的用户可见这些蓝图。

单蓝图授权规则：

- 单蓝图授权不受分组变化影响。
- 蓝图被移动分组后，单蓝图授权仍然有效。

### 9.4 MCP 服务权限

MCP 服务面向 Agent 接入有招，用于读取和发布蓝图。MCP 服务不绕过平台权限模型。

接入主体：

| 主体 | 说明 |
| --- | --- |
| Agent | 通过 MCP 客户端接入有招的外部或内部智能体。 |
| MCP Token | 分配给 Agent 的访问凭证。 |
| 绑定用户 | MCP Token 必须绑定平台用户，权限按绑定用户计算。 |

权限规则：

- Agent 获取蓝图时，必须具备绑定用户的蓝图预览查看或管理权限。
- Agent 获取蓝图时，只能访问绑定用户被授权的蓝图。
- Agent 发布蓝图时，必须具备绑定用户的蓝图预览管理权限。
- Agent 发布到指定分组时，绑定用户必须拥有该分组访问权限或系统设置管理权限。
- 系统设置管理权限不自动拥有蓝图发布权限，除非同时具备蓝图预览管理权限。
- MCP Token 可被启用、停用、删除。
- MCP Token 泄露时，管理员可以立即停用该 Token。
- Token 创建默认绑定当前登录用户，不允许在创建时选择其他用户。
- 仅拥有蓝图预览管理权限的用户可以创建 MCP Token。
- 非蓝图管理者不可创建 MCP Token，但系统设置管理者仍可停用或删除已有 Token。

MCP Token 建议字段：

| 字段 | 说明 |
| --- | --- |
| Token ID | 系统生成，唯一 |
| Token 名称 | 便于识别 Agent 来源 |
| 绑定用户 | 权限计算使用的用户 |
| 状态 | 启用、停用 |
| Token Scope | 控制 Token 可调用的能力范围 |
| 过期时间 | 可选，过期后不可使用 |
| 最近使用时间 | 用于审计 |
| 创建时间 | Token 创建时间 |

Token Scope：

| Scope | 说明 |
| --- | --- |
| `read:blueprint` | 允许读取可授权范围内的蓝图、分组和版本产物。 |
| `publish:blueprint` | 允许发布蓝图新版本。 |

权限计算规则：

- MCP Token 的最终能力 = 绑定用户权限 ∩ Token Scope。
- 绑定用户具备蓝图预览管理权限，但 Token 只有 `read:blueprint` 时，Agent 不可发布。
- Token 具备 `publish:blueprint`，但绑定用户不具备蓝图预览管理权限时，Agent 不可发布。
- Token Scope 变更后立即生效，已停用或过期 Token 不再具备任何能力。

## 10. MCP 服务

### 10.1 目标

MCP 服务用于让 Agent 以标准工具协议接入有招，完成两个核心动作：

- 获取蓝图：读取蓝图元数据、HTML 产物和 Markdown 说明。
- 发布蓝图：提交 Agent 生成的 HTML 与 Markdown，登记为新的蓝图版本并返回预览地址。

### 10.2 设计原则

- MCP 服务只暴露 Agent 所需的最小工具集合。
- MCP 服务必须复用平台用户、权限、蓝图、版本模型。
- Agent 不直接访问数据库和文件系统。
- Agent 不直接拼接预览地址，预览地址由平台返回。
- 发布动作必须生成新版本，不覆盖历史版本。
- HTML 与 Markdown 均作为蓝图版本产物保存。
- 所有 MCP 调用必须记录审计日志。

### 10.3 MCP 工具清单

MVP 阶段提供以下 MCP tools：

| Tool | 用途 | 权限要求 |
| --- | --- | --- |
| `youzhao.list_blueprints` | 查询可访问蓝图列表 | 蓝图预览查看或管理 |
| `youzhao.list_blueprint_groups` | 查询可访问蓝图分组 | 蓝图预览查看或管理 |
| `youzhao.get_blueprint` | 获取蓝图元数据 | 蓝图预览查看或管理 + 蓝图授权 |
| `youzhao.get_blueprint_artifact` | 获取指定版本产物 | 蓝图预览查看或管理 + 蓝图授权 |
| `youzhao.publish_blueprint` | 发布蓝图新版本 | 蓝图预览管理 |

工具通用约束：

- 所有工具调用必须携带 MCP Token。
- 所有读取类工具要求 Token 具备 `read:blueprint` scope。
- 发布类工具要求 Token 具备 `publish:blueprint` scope。
- 请求字段必须使用明确枚举和长度限制，未知字段允许忽略但不得影响权限判断。
- 单次 MCP 请求体默认上限 5 MB；超过上限返回 `PAYLOAD_TOO_LARGE`。
- 返回值不得包含 Token 明文、密码哈希、服务端绝对路径等敏感信息。

### 10.8 `youzhao.publish_blueprint`

用途：Agent 发布新的蓝图版本。平台保存 HTML 与 Markdown 产物，创建版本记录并返回预览地址。

请求参数：

```json
{
  "blueprintId": "blueprint_001",
  "name": "招商驾驶舱",
  "summary": "展示招商项目、线索、签约与落地情况。",
  "tags": ["招商", "驾驶舱"],
  "groupId": "group_default",
  "version": "v1.1.0",
  "html": "<!doctype html><html>...</html>",
  "markdown": "# 招商驾驶舱\n\n## 页面说明\n...",
  "publishNote": "优化项目进度模块与指标说明。",
  "idempotencyKey": "agent-job-20260624-001"
}
```

参数说明：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `blueprintId` | 否 | 传入时发布到已有蓝图；不传时创建新蓝图|
| `name` | 是 | 蓝图名称 |
| `summary` | 否 | 蓝图概述 |
| `tags` | 否 | 蓝图标签 |
| `groupId` | 否 | 不传时使用默认分组 |
| `version` | 是 | 版本号，当前蓝图内唯一 |
| `html` | 是 | 可预览 HTML 产物 |
| `markdown` | 是 | 蓝图说明文档 |
| `publishNote` | 否 | 发布说明 |
| `idempotencyKey` | 否 | 幂等键，同一发布任务重复提交时使用 |

返回结果：

```json
{
  "blueprintId": "blueprint_001",
  "versionId": "version_002",
  "version": "v1.1.0",
  "isLatest": true,
  "previewUrl": "https://example.com/preview/blueprint_001/v1.1.0/",
  "artifacts": {
    "html": true,
    "markdown": true
  },
  "publishedAt": "2026-06-12T00:00:00+08:00"
}
```

发布规则：

- `version` 在同一蓝图下必须唯一。
- 同一 `idempotencyKey` 重复提交时，应返回首次发布结果，不重复创建版本。
- 同一蓝图下已存在相同 `version`，且不是同一 `idempotencyKey`，返回 `VERSION_CONFLICT`。
- 发布成功后，新版本自动成为最新版本。
- 旧版本不删除，可继续通过版本下拉访问。
- 若 `blueprintId` 不存在，返回错误。
- 若不传 `blueprintId`，则创建新蓝图并创建首个版本。
- HTML 产物必须可生成预览入口。
- Markdown 产物用于 Agent 读取、人工评审和蓝图说明展示。
- 发布失败不得产生半完成版本记录。

产物校验规则：

- HTML 必须非空，并包含 `<!doctype html>` 或 `<html`。
- Markdown 必须非空。
- HTML 默认大小上限 2 MB。
- Markdown 默认大小上限 512 KB。
- HTML 允许内联 CSS 与 JS；外链脚本、字体、图片和接口调用在单机生产版中需走平台允许的白名单域名。
- 服务端保存前必须计算 HTML 与 Markdown 的 SHA-256 哈希。

版本状态：

| 状态 | 说明 |
| --- | --- |
| `draft` | 已创建但未对用户开放。 |
| `available` | 可预览、可读取。 |
| `disabled` | 已停用，不在默认版本列表中展示。 |
| `failed` | 发布失败或产物不可用，仅用于审计和排障。 |

产物存储清单：

```text
/data/youzhao/previews/{blueprintId}/{version}/index.html
/data/youzhao/previews/{blueprintId}/{version}/blueprint.md
/data/youzhao/previews/{blueprintId}/{version}/manifest.json
```

`manifest.json` 至少包含：

```json
{
  "blueprintId": "blueprint_001",
  "version": "v1.1.0",
  "htmlHash": "sha256:example",
  "markdownHash": "sha256:example",
  "htmlSize": 102400,
  "markdownSize": 2048,
  "publishedAt": "2026-06-12T00:00:00+08:00"
}
```

### 10.9 MCP 错误码

| 错误码 | 说明 |
| --- | --- |
| `UNAUTHENTICATED` | Token 缺失、无效或过期 |
| `FORBIDDEN` | Token 绑定用户无权限 |
| `BLUEPRINT_NOT_FOUND` | 蓝图不存在或不可访问 |
| `VERSION_NOT_FOUND` | 版本不存在或不可访问 |
| `VERSION_CONFLICT` | 版本号已存在 |
| `INVALID_ARTIFACT` | HTML 或 Markdown 产物无效 |
| `PAYLOAD_TOO_LARGE` | 请求体超过大小限制 |
| `PUBLISH_FAILED` | 发布失败 |
| `INVALID_ARGUMENT` | 请求参数不符合字段约束 |
| `SCOPE_DENIED` | Token Scope 不允许调用该工具 |
| `GROUP_NOT_FOUND` | 分组不存在或不可访问 |

### 10.10 MCP 审计日志

每次 MCP 调用必须记录：

- 调用时间。
- Request ID。
- Token ID。
- 绑定用户。
- Tool 名称。
- 请求参数摘要。
- `idempotencyKey`。
- 操作对象。
- 操作结果。
- 错误码。
- 客户端标识。
- 客户端 IP。
- 调用耗时。
- 产物大小。
- 产物哈希。

敏感内容处理：

- 不记录完整 Token。
- 不在日志中记录完整 HTML。
- 不在日志中记录完整 Markdown。
- 仅记录产物大小、哈希、版本号和对象 ID。

## 13. 关键流程

### 13.1 打开蓝图

```text
用户进入蓝图页面
→ 系统校验蓝图预览功能权限
→ 系统加载用户被授权分组和蓝图
→ 页面按分组展示蓝图卡片
→ 用户选择版本
→ 用户点击卡片
→ 系统校验版本可用
→ 新页签打开平台预览容器地址
→ 平台预览容器通过 iframe 加载蓝图 HTML 产物
→ 平台预览容器左下角展示版本与文本内容悬浮卡片
```

### 13.2 删除分组

```text
用户点击删除分组
→ 系统校验蓝图预览管理权限
→ 若为默认分组，禁止删除
→ 展示二次确认
→ 用户确认
→ 系统将该分组下蓝图移动至默认分组
→ 系统删除该分组
→ 页面刷新分组与卡片
→ 展示成功提示
```

### 13.3 调整分组顺序

```text
用户点击分组上移或下移
→ 系统校验蓝图预览管理权限
→ 若为默认分组，禁止调整
→ 系统保存新的分组顺序
→ 页面按新顺序展示分组
→ 展示成功提示
```

### 13.4 新建用户

```text
管理员进入用户管理
→ 新建用户
→ 填写用户信息
→ 提交
→ 系统创建用户
→ 系统授予蓝图预览查看权限
→ 系统授予默认分组蓝图权限
→ 返回用户列表
```

### 13.5 蓝图授权

```text
管理员进入权限管理
→ 选择用户
→ 选择蓝图权限
→ 按分组或单个蓝图勾选授权对象
→ 保存
→ 系统更新授权关系
→ 用户下次进入蓝图页面时按新权限展示
```

### 13.6 Agent 获取蓝图

```text
Agent 通过 MCP Token 调用 youzhao.get_blueprint
→ MCP 服务校验 Token 有效性
→ 系统加载 Token 绑定用户
→ 系统校验蓝图预览功能权限
→ 系统校验蓝图授权范围
→ 返回蓝图元数据和版本列表
→ 写入 MCP 审计日志
```

### 13.7 Agent 获取蓝图产物

```text
Agent 调用 youzhao.get_blueprint_artifact
→ MCP 服务校验 Token 和权限
→ 系统定位蓝图版本
→ 系统读取 html 或 markdown 产物
→ 返回产物内容和 contentType
→ 写入 MCP 审计日志
```

### 13.8 Agent 发布蓝图

```text
Agent 调用 youzhao.publish_blueprint
→ MCP 服务校验 Token 和蓝图预览管理权限
→ 系统校验蓝图、分组、版本号和产物格式
→ 系统写入 HTML 与 Markdown 产物
→ 系统创建蓝图版本记录
→ 系统更新最新版本标识
→ 系统返回预览地址
→ 写入 MCP 审计日志
```

## 14. 页面状态

蓝图页面必须覆盖：

- 加载中。
- 无蓝图预览功能权限。
- 有功能权限但无蓝图授权。
- 分组为空。
- 蓝图无可用版本。
- 版本预览地址不可用。
- 修改成功。
- 修改失败。

系统设置页面必须覆盖：

- 无系统设置管理权限。
- 用户列表为空。
- 权限保存成功。
- 权限保存失败。
- 删除用户确认。

MCP 服务必须覆盖：

- Token 缺失。
- Token 过期。
- Token 停用。
- 无蓝图查看权限。
- 无蓝图发布权限。
- 蓝图不存在。
- 版本不存在。
- 版本号冲突。
- 产物格式错误。
- 发布失败。

## 15. 非功能要求

- 权限判断必须在前端和后端同时执行。
- 前端隐藏无权限入口，后端拒绝无权限请求。
- 所有修改类操作必须有成功或失败反馈。
- 删除分组、删除用户必须二次确认。
- 蓝图预览地址必须由后端返回，不允许前端拼接。
- 蓝图卡片列表首次加载目标响应时间不超过 2 秒。
- 页面在 1440px 与 1920px 宽度下不得出现内容重叠。
- 卡片内容超长时必须截断或折行，不能撑破卡片。
- MCP 获取蓝图元数据响应目标不超过 2 秒。
- MCP 获取单个产物响应目标不超过 5 秒，超大产物后续可改为下载链接。
- MCP 发布蓝图必须具备原子性，失败不得留下半完成版本。
- MCP Token 必须支持停用。
- MCP 调用必须记录审计日志。
- HTML 与 Markdown 产物必须持久化。
- 平台统一使用莫兰迪蓝灰色系。
- 平台字体优先使用苹方：`PingFang SC`。
- 平台界面不展示“本平台为 MVP”“纯前端原型”等临时说明文案。
- 左侧菜单默认收起，仅显示图标；可通过按钮或拖拽展开。
- 平台必须使用正式 Logo，不使用临时文字标识替代。

## 16. 验收标准

### 16.1 蓝图展示

- 用户拥有蓝图预览查看权限时，可以看到左侧蓝图菜单。
- 用户点击蓝图菜单后，页面以卡片形式显示已授权蓝图。
- 蓝图卡片展示名称、概述、标签和版本下拉。
- 蓝图卡片默认选择最新版本。
- 用户切换版本后，点击卡片打开对应版本的新页签。
- 用户无某蓝图权限时，该蓝图不展示。

### 16.2 分组管理

- 蓝图管理者可以新建分组。
- 蓝图管理者可以删除非默认分组。
- 默认分组不可删除。
- 删除分组后，该分组下蓝图移动至默认分组。
- 蓝图管理者可以通过上移、下移调整非默认分组顺序。
- 默认分组固定置顶，不参与排序。
- 蓝图管理者可以拖动卡片改变分组。
- 蓝图管理者可以通过右键菜单修改蓝图信息和分组。
- 蓝图查看者不能新建分组、删除分组、拖动卡片或修改卡片信息。

### 16.3 用户管理

- 系统设置管理者可以新建用户。
- 新建用户默认拥有蓝图预览查看权限。
- 新建用户默认拥有默认分组蓝图权限。
- 系统设置管理者可以修改除用户 ID 外的用户信息。
- 系统设置管理者可以删除用户。

### 16.4 权限管理

- 系统设置管理者可以为用户配置蓝图预览查看或管理权限。
- 系统设置管理者可以为用户配置系统设置管理权限。
- 系统设置管理者可以按分组授权蓝图。
- 系统设置管理者可以按单个蓝图授权蓝图。
- 用户最终可见蓝图等于分组授权与单蓝图授权的并集。

### 16.5 MCP 服务

- Agent 可以通过 `youzhao.list_blueprints` 获取绑定用户可访问的蓝图列表。
- Agent 无蓝图权限时，无法获取未授权蓝图。
- Agent 可以通过 `youzhao.get_blueprint` 获取蓝图元数据和版本列表。
- Agent 可以通过 `youzhao.get_blueprint_artifact` 获取指定版本 HTML。
- Agent 可以通过 `youzhao.get_blueprint_artifact` 获取指定版本 Markdown。
- Agent 拥有蓝图预览管理权限时，可以通过 `youzhao.publish_blueprint` 发布新版本。
- Agent 发布成功后，蓝图卡片版本下拉中可以看到新版本。
- Agent 发布成功后，平台返回可打开的预览地址。
- Agent 使用重复版本号发布时，返回 `VERSION_CONFLICT`。
- MCP Token 停用后，Agent 无法继续调用工具。
- 每次 MCP 调用均产生审计日志。
- 空分组创建后立即可见。
- 卡片拖动到原分组时不提示分组变更。
- Token 创建默认绑定当前登录用户。
- 非蓝图管理者无法创建 Token。

## 17. 待确认问题

- 用户登录与认证方式是否本期实现，还是先用本地模拟用户。
- 蓝图数据是否本期支持手工录入，还是先使用后端种子数据。
- 蓝图管理权限是否允许新增蓝图，当前需求仅覆盖修改展示信息和分组。
- 删除用户采用硬删除还是停用优先。
- 蓝图卡片是否需要支持关键词搜索和标签筛选。
- 版本选择是否需要记住用户上次选择。
- MCP Token 是否在本期提供系统设置页面管理入口。
- MCP 发布蓝图是否允许创建新分组，还是只能发布到已有分组。
- MCP HTML 产物大小上限是多少。
- MCP Markdown 是否需要进入人工评审流程后再展示。
