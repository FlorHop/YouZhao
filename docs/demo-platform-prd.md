# 有招平台 Demo 展示功能 PRD

## 1. 文档信息

- 产品名称：有招平台
- 模块范围：Demo 展示、系统设置、用户管理、权限管理
- 文档版本：v0.1
- 当前阶段：MVP 功能设计
- 前端约束参考：[前端统一规则约束](./frontend-rules.md)

## 2. 背景与目标

有招平台需要先从“已部署 Demo 的展示、授权、预览”开始实践 MVP。平台面向内部交付、售前、业务演示等场景，用户可以在统一入口查看已部署 Demo，并按权限打开对应版本的 Demo。

本阶段目标：

- 提供统一 Demo 入口。
- 支持 Demo 按分组展示和管理。
- 支持 Demo 卡片基础信息展示。
- 支持 Demo 版本选择和新页签预览。
- 支持用户管理。
- 支持功能权限和 Demo 访问权限管理。
- 明确查看权限与管理权限的操作边界。

## 3. 用户角色

| 角色 | 说明 |
| --- | --- |
| 平台管理员 | 拥有系统设置管理权限，可管理用户、功能权限、Demo 权限。 |
| Demo 管理者 | 拥有 Demo 预览管理权限，可维护 Demo 分组、卡片信息、卡片归属分组。 |
| Demo 查看者 | 拥有 Demo 预览查看权限，可查看被授权的 Demo，选择版本并打开预览。 |

说明：

- MVP 阶段不单独设计角色模型，权限直接授权到用户。
- 后续可扩展为“用户 - 角色 - 权限”的 RBAC 模型。

## 4. 功能范围

### 4.1 本期范围

- 左侧功能栏新增“Demo”菜单。
- Demo 页面以卡片形式展示已部署 Demo。
- Demo 卡片按分组展示。
- Demo 页面支持新建、删除分组。
- 删除分组时，将该分组下卡片移动至“默认”分组。
- “默认”分组不可删除。
- Demo 卡片展示名称、概述、标签。
- Demo 卡片支持拖动改变分组。
- Demo 卡片支持右键菜单修改卡片信息和分组。
- Demo 卡片支持版本下拉选择，默认最新版本。
- 点击 Demo 卡片在浏览器新页签打开所选版本 Demo。
- 系统设置包含用户管理和权限管理。
- 用户管理支持新建、修改、删除。
- 权限管理支持功能权限授权。
- 权限管理支持按分组或单个 Demo 授权。
- 新建用户默认拥有 Demo 预览查看权限。
- 新建用户默认拥有“默认”分组的 Demo 展示权限。

### 4.2 暂不纳入本期

- Demo 上传、构建、发布流水线。
- Demo 运行态内的细粒度页面权限。
- 角色管理。
- 组织架构管理。
- 单点登录。
- 审批流。
- 操作审计详情页。
- Demo 使用统计分析。
- Demo 全文搜索。

## 6. 导航与页面

### 6.1 左侧功能栏

左侧功能栏包含：

| 菜单 | 显示条件 | 路由建议 |
| --- | --- | --- |
| Demo | 用户拥有 Demo 预览查看或管理权限 | `/demo` |
| 系统设置 | 用户拥有系统设置管理权限 | `/settings` |

菜单规则：

- 用户没有任何可访问菜单时，显示无权限页。
- 系统设置仅有管理权限，不提供查看权限。
- Demo 菜单内的数据仍需按 Demo 权限过滤。

### 6.2 Demo 页面

页面目标：让用户快速查看、选择版本并打开已授权 Demo。

页面布局：

```text
PageHeader: Demo
Toolbar: 新建分组 / 刷新 / 视图设置
ContentRegion:
  GroupSection: 默认
    DemoCard[]
  GroupSection: 其他分组
    DemoCard[]
```

显示规则：

- 仅展示当前用户被授权的分组和 Demo。
- 分组为空时仍可展示分组标题和空状态。
- 默认分组始终存在。
- Demo 卡片按分组展示。
- 分组顺序 MVP 阶段可按创建时间排序，默认分组置顶。

管理能力：

- 拥有 Demo 预览管理权限时，显示新建分组入口。
- 拥有 Demo 预览管理权限时，显示分组删除入口。
- 拥有 Demo 预览管理权限时，允许拖动卡片改变分组。
- 拥有 Demo 预览管理权限时，允许通过右键菜单修改卡片内容和分组。
- 仅拥有 Demo 预览查看权限时，不允许修改分组、名称、概述、标签。

查看能力：

- 拥有 Demo 预览查看权限时，可以查看已授权 Demo。
- 版本下拉选择不属于数据修改类操作，查看权限用户也可使用。
- 点击卡片可新页签打开当前选择版本。

### 6.3 系统设置页面

系统设置包含两个模块：

- 用户管理
- 权限管理

显示规则：

- 仅拥有系统设置管理权限的用户可访问。
- 未授权用户访问时显示无权限页。

## 7. Demo 展示功能

### 7.1 Demo 卡片

卡片字段：

| 字段 | 说明 | 是否必填 |
| --- | --- | --- |
| Demo 名称 | 当前 Demo 的展示名称 | 是 |
| 概述 | Demo 的简短说明 | 否 |
| 标签 | 用于标识行业、场景、技术、状态等 | 否 |
| 版本 | 当前可预览版本列表 | 是 |
| 当前选择版本 | 卡片当前选中的版本 | 是 |

卡片操作：

| 操作 | 权限要求 | 说明 |
| --- | --- | --- |
| 选择版本 | Demo 预览查看或管理 | 不修改 Demo 数据，仅影响本次打开版本。 |
| 点击打开 | Demo 预览查看或管理 + Demo 授权 | 新页签打开所选版本 Demo。 |
| 拖动到分组 | Demo 预览管理 | 修改 Demo 所属分组。 |
| 右键编辑信息 | Demo 预览管理 | 修改名称、概述、标签。 |
| 右键移动分组 | Demo 预览管理 | 修改 Demo 所属分组。 |

交互规则：

- 卡片点击区域用于打开 Demo。
- 版本下拉点击不触发卡片打开。
- 右键卡片打开上下文菜单，不触发卡片打开。
- 拖动卡片时不触发卡片打开。
- 修改成功后展示 Toast。
- 修改失败时展示 Toast，并保持原状态。

### 7.2 版本选择

版本规则：

- 每个 Demo 至少有一个已部署版本。
- 默认选中最新版本。
- 最新版本以版本创建时间或部署时间倒序确定。
- 用户切换版本后，只影响当前页面会话中的选中值。
- 刷新页面后重新默认最新版本，除非后续实现用户偏好保存。

打开规则：

- 点击卡片时，根据当前选中版本获取预览地址。
- 预览地址必须在新浏览器页签打开。
- 若版本不可用，提示“当前版本不可用”，不打开新页签。

### 7.3 标签

标签用于提升识别效率。

规则：

- 标签展示在卡片正文区域。
- 单张卡片最多显示 5 个标签。
- 超出标签折叠为 `+N`。
- 标签内容由 Demo 管理者维护。

### 7.4 分组管理

分组字段：

| 字段 | 说明 |
| --- | --- |
| 分组 ID | 系统生成，唯一 |
| 分组名称 | 页面展示名称 |
| 是否默认分组 | 标识默认分组 |
| 创建时间 | 分组创建时间 |

新建分组：

- 入口：Demo 页面 Toolbar。
- 权限：Demo 预览管理。
- 分组名称必填。
- 分组名称在当前租户或平台范围内不可重复。
- 新建成功后分组出现在 Demo 页面。

删除分组：

- 入口：分组标题区域。
- 权限：Demo 预览管理。
- 默认分组不可删除。
- 删除非默认分组时需要二次确认。
- 删除后，该分组下所有 Demo 卡片移动至“默认”分组。
- 删除成功后展示 Toast。

删除确认文案：

```text
删除分组后，该分组下的 Demo 将移动至“默认”分组。是否继续？
```

## 8. 用户管理

用户列表字段：

| 字段 | 说明 |
| --- | --- |
| 用户 ID | 系统主键，不可修改 |
| 用户名 | 登录或展示名称 |
| 姓名 | 用户真实姓名 |
| 邮箱 | 联系邮箱 |
| 手机号 | 联系手机号 |
| 状态 | 启用、停用 |
| 创建时间 | 用户创建时间 |

功能：

| 功能 | 说明 |
| --- | --- |
| 新建用户 | 创建用户，并自动赋予默认权限。 |
| 修改用户 | 可修改除用户 ID 外的信息。 |
| 删除用户 | 删除用户及其权限关系。 |

新建用户默认权限：

- 开放 Demo 预览功能的查看权限。
- 开放 Demo 展示功能中的“默认”分组权限。

删除规则：

- 删除用户需要二次确认。
- 删除用户后，该用户不可再登录或访问平台。
- MVP 阶段可采用硬删除；如后续需要审计，改为停用优先。

## 9. 权限管理

### 9.1 权限模型

MVP 阶段采用用户直接授权模型。

```text
User
├─ FunctionPermission[]
└─ DemoPermission[]
```

### 9.2 功能权限

功能板块：

| 功能板块 | 查看 | 管理 |
| --- | --- | --- |
| Demo 预览 | 支持 | 支持 |
| 系统设置 | 不支持 | 支持 |

权限说明：

- Demo 预览查看：可访问 Demo 页面、查看已授权 Demo、选择版本、打开 Demo。
- Demo 预览管理：包含查看能力，并可新建/删除分组、修改 Demo 信息、修改 Demo 分组。
- 系统设置管理：可访问系统设置、管理用户、管理功能权限、管理 Demo 权限。

查看权限不可执行的数据修改类操作：

- 新建分组。
- 删除分组。
- 拖动 Demo 卡片改变分组。
- 通过右键菜单修改 Demo 名称、概述、标签。
- 通过右键菜单修改 Demo 所属分组。

查看权限允许执行的非数据修改操作：

- 选择 Demo 版本。
- 打开 Demo。

### 9.3 Demo 权限

授权形式：

| 授权方式 | 说明 |
| --- | --- |
| 按分组授权 | 用户可访问该分组下 Demo。 |
| 按单个 Demo 授权 | 用户可访问指定 Demo。 |

权限合并规则：

- 用户最终可见 Demo = 已授权分组下 Demo ∪ 已授权单个 Demo。
- 若 Demo 同时被分组授权和单独授权，不重复展示。
- 用户有 Demo 预览功能权限，但无任何 Demo 权限时，进入 Demo 页面显示空状态。

分组授权规则：

- 新建用户默认授权“默认”分组。
- 删除分组后，原分组授权关系失效。
- 被删除分组下 Demo 移动至默认分组后，拥有默认分组权限的用户可见这些 Demo。

单 Demo 授权规则：

- 单 Demo 授权不受分组变化影响。
- Demo 被移动分组后，单 Demo 授权仍然有效。

## 12. 关键流程

### 12.1 打开 Demo

```text
用户进入 Demo 页面
→ 系统校验 Demo 预览功能权限
→ 系统加载用户被授权分组和 Demo
→ 页面按分组展示 Demo 卡片
→ 用户选择版本
→ 用户点击卡片
→ 系统校验版本可用
→ 新页签打开版本预览地址
```

### 12.2 删除分组

```text
用户点击删除分组
→ 系统校验 Demo 预览管理权限
→ 若为默认分组，禁止删除
→ 展示二次确认
→ 用户确认
→ 系统将该分组下 Demo 移动至默认分组
→ 系统删除该分组
→ 页面刷新分组与卡片
→ 展示成功提示
```

### 12.3 新建用户

```text
管理员进入用户管理
→ 新建用户
→ 填写用户信息
→ 提交
→ 系统创建用户
→ 系统授予 Demo 预览查看权限
→ 系统授予默认分组 Demo 权限
→ 返回用户列表
```

### 12.4 Demo 授权

```text
管理员进入权限管理
→ 选择用户
→ 选择 Demo 权限
→ 按分组或单个 Demo 勾选授权对象
→ 保存
→ 系统更新授权关系
→ 用户下次进入 Demo 页面时按新权限展示
```

## 13. 页面状态

Demo 页面必须覆盖：

- 加载中。
- 无 Demo 预览功能权限。
- 有功能权限但无 Demo 授权。
- 分组为空。
- Demo 无可用版本。
- 版本预览地址不可用。
- 修改成功。
- 修改失败。

系统设置页面必须覆盖：

- 无系统设置管理权限。
- 用户列表为空。
- 权限保存成功。
- 权限保存失败。
- 删除用户确认。

## 14. 非功能要求

- 权限判断必须在前端和后端同时执行。
- 前端隐藏无权限入口，后端拒绝无权限请求。
- 所有修改类操作必须有成功或失败反馈。
- 删除分组、删除用户必须二次确认。
- Demo 预览地址必须由后端返回，不允许前端拼接。
- Demo 卡片列表首次加载目标响应时间不超过 2 秒。
- 页面在 1440px 与 1920px 宽度下不得出现内容重叠。
- 卡片内容超长时必须截断或折行，不能撑破卡片。

## 15. 验收标准

### 15.1 Demo 展示

- 用户拥有 Demo 预览查看权限时，可以看到左侧 Demo 菜单。
- 用户点击 Demo 菜单后，页面以卡片形式显示已授权 Demo。
- Demo 卡片展示名称、概述、标签和版本下拉。
- Demo 卡片默认选择最新版本。
- 用户切换版本后，点击卡片打开对应版本的新页签。
- 用户无某 Demo 权限时，该 Demo 不展示。

### 15.2 分组管理

- Demo 管理者可以新建分组。
- Demo 管理者可以删除非默认分组。
- 默认分组不可删除。
- 删除分组后，该分组下 Demo 移动至默认分组。
- Demo 管理者可以拖动卡片改变分组。
- Demo 管理者可以通过右键菜单修改 Demo 信息和分组。
- Demo 查看者不能新建分组、删除分组、拖动卡片或修改卡片信息。

### 15.3 用户管理

- 系统设置管理者可以新建用户。
- 新建用户默认拥有 Demo 预览查看权限。
- 新建用户默认拥有默认分组 Demo 权限。
- 系统设置管理者可以修改除用户 ID 外的用户信息。
- 系统设置管理者可以删除用户。

### 15.4 权限管理

- 系统设置管理者可以为用户配置 Demo 预览查看或管理权限。
- 系统设置管理者可以为用户配置系统设置管理权限。
- 系统设置管理者可以按分组授权 Demo。
- 系统设置管理者可以按单个 Demo 授权 Demo。
- 用户最终可见 Demo 等于分组授权与单 Demo 授权的并集。
